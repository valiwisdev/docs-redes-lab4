## Descripción

En esta topología:

- `R2` actúa como router central.
- `R2` se conecta con `R1`, `R3` y una LAN propia.
- Se manejan tres sistemas autónomos: `ASN 100`, `ASN 200` y `ASN 300`.

## Comandos para identificar las interfaces de red

```bash
show ip interface brief
show interfaces description
show running-config | section interface
```

## Comandos útiles

Estos comandos ayudan a revisar, corregir o limpiar la configuración durante las pruebas con `BGP`.

| Acción | Comando de ejemplo |
| --- | --- |
| Guardar configuración | `copy running-config startup-config` |
| Ver tabla de rutas | `show ip route` |
| Ver vecinos BGP | `show ip bgp summary` |
| Reiniciar una interfaz | `interface g0/1`<br/>`shutdown`<br/>`no shutdown` |
| Restaurar una interfaz a valores por defecto | `default interface g0/1` |
| Quitar proceso BGP en `ASN 100` | `no router bgp 100` |
| Quitar proceso BGP en `ASN 200` | `no router bgp 200` |
| Quitar proceso BGP en `ASN 300` | `no router bgp 300` |
| Reiniciar el router | `reload` |
| Borrar la configuración guardada y reiniciar | `erase startup-config`<br/>`reload` |

> Cambie `g0/1` por la interfaz real del router. El número de AS debe coincidir con el equipo que esté configurando.
