## Descripcion

En esta topologia:

- `R2` actua como router central.
- `R2` se conecta con `R1`, `R3` y una LAN propia.
- Se manejan tres sistemas autonomos: `ASN 100`, `ASN 200` y `ASN 300`.


## Comandos utiles

Estos comandos ayudan a revisar, corregir o limpiar la configuracion durante las pruebas con `BGP`.

| Accion | Comando de ejemplo |
| --- | --- |
| Guardar configuracion | `copy running-config startup-config` |
| Ver tabla de rutas | `show ip route` |
| Ver vecinos BGP | `show ip bgp summary` |
| Reiniciar una interfaz | `interface g0/1`<br/>`shutdown`<br/>`no shutdown` |
| Restaurar una interfaz a valores por defecto | `default interface g0/1` |
| Quitar proceso BGP en `ASN 100` | `no router bgp 100` |
| Quitar proceso BGP en `ASN 200` | `no router bgp 200` |
| Quitar proceso BGP en `ASN 300` | `no router bgp 300` |
| Reiniciar el router | `reload` |
| Borrar la configuracion guardada y reiniciar | `erase startup-config`<br/>`reload` |

> Cambie `g0/1` por la interfaz real del router. El numero de AS debe coincidir con el equipo que este configurando.
